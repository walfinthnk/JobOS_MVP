# JobOS MVP — 要件定義書

## 1. 概要

転職活動を支援する Web アプリケーション。求人応募の進捗管理と Gmail 連携による自動ステータス追跡を提供する。

---

## 2. 機能要件

### 2-1. 認証

| ID | 要件 |
|---|---|
| AUTH-01 | メールアドレス・パスワードでのサインアップ／ログイン |
| AUTH-02 | Google OAuth によるログイン |
| AUTH-03 | パスワードリセット（メール送信） |
| AUTH-04 | ログアウト |

### 2-2. 求人管理

| ID | 要件 |
|---|---|
| JOB-01 | 求人の新規登録（企業名・職種・URL・ステータス・応募日・メモ） |
| JOB-02 | 求人の編集・削除 |
| JOB-03 | ステータス変更（applied / screening / interview / offered / accepted / declined） |
| JOB-04 | ステータス変更履歴の自動記録 |
| JOB-05 | メモの自動保存（500ms デバウンス） |

### 2-3. ダッシュボード

| ID | 要件 |
|---|---|
| DASH-01 | Kanban ボード表示（応募中・書類選考・面接中・内定の4列） |
| DASH-02 | 辞退・不採用の折りたたみ表示 |

### 2-4. 統計

| ID | 要件 |
|---|---|
| STAT-01 | 応募数・書類通過率・面接通過率・活動日数の表示 |
| STAT-02 | ステータス別件数の表示 |
| STAT-03 | 月別応募数の棒グラフ |

### 2-5. Gmail 連携

| ID | 要件 |
|---|---|
| GMAIL-01 | Google OAuth による Gmail 連携（接続・切断） |
| GMAIL-02 | Pub/Sub Push 通知によるリアルタイム同期 |
| GMAIL-03 | Gmail連携ページのリロード時に同期を自動実行する |
| GMAIL-04 | 「Gmail同期」ボタンのクリックで手動同期を実行する |
| GMAIL-05 | メール解析による企業名・ステータス自動検出 |
| GMAIL-06 | 信頼度 70% 未満のメールを pending_review として手動確認フローへ |
| GMAIL-07 | 同期ログ一覧表示（件名・企業名・検出ステータス・確信度・アクション） |
| GMAIL-08 | pending_review の手動確認（ステータス確定・スキップ） |
| GMAIL-09 | Gmail watch() の自動更新（Vercel Cron、毎日 01:00 UTC） |

### 2-6. UI・ナビゲーション

| ID | 要件 |
|---|---|
| UI-01 | ナビメニューの Gmail 連携リンクのラベルを「Gmail連携」とする |
| UI-02 | Gmail 連携ページのタイトルを「Gmail連携」とする |

---

## 3. 非機能要件

| 項目 | 内容 |
|---|---|
| セキュリティ | RLS によるユーザーデータ分離、AES-256-GCM によるトークン暗号化 |
| 認証 | Supabase Auth（メール＋Google OAuth） |
| インフラ | Vercel（Next.js 14 App Router）+ Supabase（PostgreSQL） |
| 外部連携 | Google Gmail API、Google Cloud Pub/Sub |
